# Bedarfsforderung konkretisieren — Dialogpfad A (Unterjährig)

**Quelle:** § 6 BHO (Funktionale Bedarfsforderung)

---

## Überblick: Die 4 Anforderungen

Eine gültige Bedarfsforderung muss:

1. **Ergebnisoffen** — nicht: „Drucker kaufen" (schreibt Lösung vor)
   - Richtig: „Druckkapazität zur Unterstützung der Büroprozesse"

2. **Lösungsneutral** — nicht: „Multifunktionsdrucker" (nennt Produkttyp)
   - Richtig: „Geräte mit Druck- und Kopierfunktion" (nennt nur Funktionen)

3. **Funktional** — nicht: „Ausgaben reduzieren" (zu abstrakt)
   - Richtig: „Fähigkeit, 150 Seiten pro Stunde zu verarbeiten" (konkrete Funktion mit Maßstab)

4. **Messbar qualitativ UND quantitativ**
   - Qualitativ: Anforderungen an das WAS (z.B. Farbdruck, Netzwerkanbindung)
   - Quantitativ: Anforderungen an das WIE VIEL (z.B. Seitenleistung, Kapazität)

---

## Transformation — User Input → Bedarfsforderung

### Schritt 1: User nennt, was gekauft werden soll

**User sagt:** „Wir brauchen einen neuen Drucker für die Büros."

**FEHLER — nicht verwenden:**
> „Die Dienststelle benötigt einen Multifunktionsdrucker."
> (Problem: nennt Produkttyp statt Funktion)

---

### Schritt 2: Funktionale Merkmale extrahieren

**Fragen an User stellen:**
- Welche Funktionen braucht das Gerät? (Drucken, Kopieren, Scannen, Fax?)
- Für wie viele Nutzer / wie viele Seiten pro Monat?
- Welche Ausgabequalität? (s/w, Farbe? Auflösung?)
- Besondere Anforderungen? (Netzwerk, Sicherheit, Papierformate, Geschwindigkeit?)

**User antwortet:**
- „Druckfunktion für Bürodokumente"
- „Kopierfunktion"
- „Etwa 10.000 Seiten pro Monat"
- „Farbdruck ist wichtig"
- „Netzwerkanbindung erforderlich"

---

### Schritt 3: Bedarfsforderung formulieren (Satzmuster)

**RICHTIG — verwenden:**

> „Die [Dienststelle] benötigt eine **Druckfunktionalität mit Kopierfunktion** zur Unterstützung der Büroprozesse. Der Bedarf umfasst **eine Kapazität von mindestens 10.000 Seiten pro Monat** mit folgenden Mindestanforderungen:
> - Ausgabequalität: Farbdruck (CMYK)
> - Netzwerkanbindung (Ethernet/WLAN)
> - Maximale Verarbeitungszeit pro Seite: 5 Sekunden
> - Aufstellort: Büroraum Sachbearbeiter
>
> Das Gerät ist voraussichtlich täglich einzusetzen, während des Regelbetriebs (Mo–Fr, 08:00–17:00 Uhr)."

**Merkmale dieses Textes:**
- ✅ Ergebnisoffen: nennt nicht „Multifunktionsdrucker", „Kyocera", „Brother", etc.
- ✅ Lösungsneutral: beschreibt nur Funktionen (Druck, Kopie, Farbe, Netzwerk)
- ✅ Funktional: „Druckfunktionalität mit Kopierfunktion" ist konkrete Funktion, nicht Produktname
- ✅ Messbar qualitativ: Farbdruck, Netzwerkanbindung, Verarbeitungszeit
- ✅ Messbar quantitativ: 10.000 Seiten/Monat, 5 Sekunden/Seite

### AUTO-GENERIERUNG (NUR mit tatsächlichen Antworten)

**Pseudo-Code:**
```python
def generiere_bedarfsforderung(antworten):
    """
    Generiert Bedarfsforderung NUR aus tatsächlichen Antworten.
    KEINE erfundenen Details!
    """
    
    # Funktionen auflisten
    funktionen = ", ".join(antworten['funktionen'])  # z.B. "Druck, Kopie, Scan"
    
    # Kapazität mit Zeitspanne
    kapazitaet = f"{antworten['menge']} Seiten/{antworten['zeitspanne']}"
    
    # Qualitative Anforderungen (aus tatsächlichen Antworten)
    anforderungen = []
    if antworten['farbe'] == 'CMYK':
        anforderungen.append('Farbdruck (CMYK)')
    elif antworten['farbe'] == 'gelegentlich':
        anforderungen.append('Gelegentlicher Farbdruck')
    else:
        anforderungen.append('Schwarzweiß-Druck')
    
    if antworten['netzwerk'] in ['erforderlich', 'optional']:
        anforderungen.append(f"Netzwerkanbindung ({antworten['netzwerk']})")
    
    if antworten['duplex']:
        anforderungen.append("Beidseitiger Druck (Duplex)")
    if antworten['a3']:
        anforderungen.append("Großformat-Fähigkeit (DIN A3)")
    if antworten['geschwindigkeit']:
        anforderungen.append(f"Druckgeschwindigkeit: {antworten['geschwindigkeit']}")
    
    # Einsatzhäufigkeit mit konkreten Angaben
    if antworten['haeufigkeit'] == 'täglich_mo_fr':
        einsatzhaeufigkeit = f"täglich während der Regelarbeitszeit (Mo–Fr, {antworten['einsatztage_jahr']} Tage/Jahr)"
    elif antworten['haeufigkeit'] == 'täglich_24_7':
        einsatzhaeufigkeit = f"täglich inkl. Wochenenden/Schichtbetrieb ({antworten['einsatztage_jahr']} Tage/Jahr)"
    else:
        einsatzhaeufigkeit = f"{antworten['haeufigkeit']} ({antworten['einsatztage_jahr']} Tage/Jahr)"
    
    # SATZMUSTER zusammenstellen (NUR mit echten Daten)
    text = f"""Die [Dienststelle] benötigt eine {funktionen}fähigkeit zur Unterstützung der Büroprozesse. Der Bedarf umfasst eine Kapazität von mindestens {kapazitaet} mit folgenden Mindestanforderungen: {', '.join(anforderungen)}. Das Gerät ist voraussichtlich {einsatzhaeufigkeit} einzusetzen."""
    
    return text
```

## Checkliste: Ist die Bedarfsforderung gültig?

**VOR dem Export prüfen:**

- [ ] Enthält **keine** Produktnamen (Drucker, Laptop, VW, etc.)?
- [ ] Nennt nur **Funktionen** (Druckfunktion, Transportfähigkeit, Sitzmöbel)?
- [ ] Enthält **mindestens 3 messbare Anforderungen** (quantitativ oder qualitativ)?
- [ ] Nennt **Kapazität oder Häufigkeit** (10.000 Seiten/Monat, 15.000 km/Jahr)?
- [ ] Erklärt, **wofür** die Funktion benötigt wird (Büroprozesse, Feldbeamte, Logistik)?
- [ ] Nennt **Einsatzhäufigkeit** (täglich, 2x pro Woche, saisonal)?

**Anpassung erforderlich, wenn:**
- ❌ User „Drucker" sagt → formuliere als „Druckfunktion"
- ❌ User „Multifunktionsdrucker" sagt → nenne nur Funktionen, nicht das Produkt
- ❌ Keine Mengenangabe → frage nach Kapazität oder Häufigkeit
- ❌ Kein Einsatzzweck → ergänze, wofür die Leistung gebraucht wird

## Integration in Python (Skill-Code)

```python
def konkretisiere_bedarfsforderung(user_sachverhalt: str) -> dict:
    """
    Transformiert User-Input in BHO-konforme Bedarfsforderung.
    
    Args:
        user_sachverhalt: Was der User sagt, z.B. "Wir brauchen einen Drucker"
    
    Returns:
        {
            'funktionen': ['Druckfunktion', 'Kopierfunktion'],
            'kapazitaet': '10.000 Seiten/Monat',
            'qualitative_anforderungen': [...],
            'einsatzhaeufigkeit': 'täglich',
            'bedarfsforderung_text': '[vollständiger Satzmuster-Text]'
        }
    """
    
    # Schritt 1: Fragen stellen → Dialog
    dialog_fragen = [
        "Was soll das Gerät/Material können?",
        "Wie viele Einheiten/Mengen pro Monat?",
        "Qualitative Anforderungen?",
        "Einsatzhäufigkeit?",
    ]
    
    # Schritt 2: Antworten sammeln
    user_antworten = frage_user(dialog_fragen)
    
    # Schritt 3: Satzmuster füllen
    bedarfsforderung = satzmuster_bedarfsforderung.format(
        dienststelle=meta['dienststelle'],
        funktionen=user_antworten['funktionen'],
        kapazitaet=user_antworten['kapazitaet'],
        anforderungen=user_antworten['anforderungen'],
        einsatz=user_antworten['einsatzhaeufigkeit'],
    )
    
    # Schritt 4: Validierung
    validierung = validate_step('bedarfsforderung', bedarfsforderung)
    if validierung['status'] == '❌':
        # Rückkopplung an User für Korrektur
        return handle_validation_error(validierung)
    
    return {
        'bedarfsforderung_text': bedarfsforderung,
        'validierung': validierung,
    }
```

| FEHLER | BEISPIEL | RICHTIG |
|---|---|---|
| Produktname statt Funktion | „Multifunktionsdrucker" | „Druckfunktion mit Kopierfunktion" |
| Zu abstrakt | „Büroausstattung" | „Druckfunktionalität … mit Kapazität von 10.000 S./Monat" |
